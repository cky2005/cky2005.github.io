我部署的时候遇到的问题:

默认情况下，Heroku使用HTTP作为Git传输。Heroku CLI将自动将凭证放入.netrc文件中heroku login。Git客户端在与HTTP远程数据库交互时使用cURL，cURL将使用该.netrc文件中的凭据。有关详细信息，请参阅身份验证部分和CLI验证文章。
git push的时候会出现需要输入用户名和密码,

输入错误就会出现下面提示:

Heroku HTTP Git端点仅接受基于API密钥的HTTP Basic身份验证.不需要用户名,忽略任何传递的用户名.

解决方法:
提示输入用户名就直接回车忽略,密码就输入:API密钥(密匙可以去heroku网站用户设置哪里查看或生成)

$$-heroku(这里的appname是你的app名字)
