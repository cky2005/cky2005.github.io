hexo配置文件_config.yml蛋疼的空格问题...

看教程部署hexo到github
刚开始用notepad++编辑保存 _config.yml

运行hexo clean 提示 type位置出错,ok,试着删除type/repo/bbranch前面的2个空格:

hexo clean运行成功
然后运行 hexo -g 成功
再运行 hexo -d 什么提示都没有 运行失败
网上搜了搜 都是说什么冒号后面要有空格才行,我看看我的配置文件,冒号后面有空格啊,反复实验了n次仍然失败

想了想,莫非是保存文件的编码问题,把原来的hexo目录删除,重新hexo init了一次
这次用自带的vi编辑保存 _config.yml
因为前面的有空格出现错误,这次直接也删除掉type/repo/bbranch前面的2个空格
运行hexo -d 依然什么提示都没有 运行失败
实验了n次失败后,决定把type/repo/bbranch前面的2个空格加回去
运行hexo -d 这次成功了
