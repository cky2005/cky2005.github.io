    github-pages 自定义二级域名加https

    方法是通过免费不限流量CDN Cloudflare 实现.

    示例,用的是二级域名blog.233666.org绑定到github-pages

    方法(1)在github-pages仓库新建一个CNAME文件,里面的内容为你的二级域名.
    方法(2)在你的仓库github-pages页设置自定义域名.

    去你的域名商设置哪里,域名dns新增一个项目:

    子域名(Name)记录类型(Type)ip地址/目标主机填写(Value)
    blogCNAME你的github-pages.github.io

    然后去Cloudflare注册一个账号

    加你的网站进去,譬如我加的是233666.org

    去你的域名服务商 将 域名服务器设置为Cloudflare提供的2个域名服务器:

    TypeValue
    NSart.ns.cloudflare.com
    NSelsa.ns.cloudflare.com

    稍微等一会儿等设置生效.

    回到cloudflare的overview设置页,查看状态是否active

    cloudflare - dns设置页

    NAME记录类型(Type)ip地址/目标主机填写status
    233666.orgA服务器ip地址DNS only
    wwwA服务器ip地址DNS only
    blogCNAME你的github-pages.github.ioDNS and HTTP proxy(CDN)

    我的233666.org和www, Status 选择的是DNS only,跳过了cdn,只有blog.233666.org设置了走cdn.

    cloudflare - cypto设置页

    ssl 设置为 flexible , Automatic HTTPS Rewrites 设置为 on
    (设置后,cloudflare会为你自动申请一个证书,要等上一段时间才能生效)

    cloudflare - Page Rules设置页

    create page rules设置http:/blog.xxx.xxx/*全部走https通道

    注意:设置生效可能要等上一段时间的,不是一设置就立即生效的.

    示例地址: https://blog.233666.org

    有时候这个dns解析真的很混乱,不知道什么时候生效或者不知道设置正不正确,很可能搞到网站短时间内访问出错…
