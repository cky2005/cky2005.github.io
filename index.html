搭建hexo博客

搭建流程

第一步： 安装node.js以及本地Hexo初始化
第二步： 服务器环境搭建，包括安装 Git 、Nginx配置 、创建 git 用户

第三步： 使用Git自动化部署发布博客

本地环境

安装node.js

初始化Hexo博客

首先创建 你的Hexo 目录

然后安装 hexo-cli，安装hexo-cli 需要 root 权限，使用 sudo 运行

初始化该文件夹

安装hexo的依赖,hexo是基于nodejs开发的，npm是nodejs的包管理工具

安装两个插件， hexo-deployer-git 和 hexo-server ,这俩插件的作用分别是使用Git自动部署，和本地简单的服务器。

以及一些其他插件（*非必选）
npm install hexo-admin –save
npm install hexo-generator-archive –save
npm install hexo-generator-feed –save
npm install hexo-generator-search –save
npm install hexo-generator-tag –save
npm install hexo-generator-sitemap –save

执行该命令后在hexo目录下的 sources/_posts文件夹里生成了刚刚创建的hello

Hexo.md的markdown文件，然后就可以通过本地或者在线的markdown编辑器就可以创作自己的博客了~

编辑完毕以后， 使用hexo g将 .md文件渲染成静态文件，然后启动hexo-server

用浏览器打开 http://localhost:4000 如果看到 hexo 的初始页面证明安装成功。

这个公钥将会复制到服务器的证书中，添加公钥之后可以防止每次 push 都输入密码。

服务器环境搭建

安装node.js;git;创建一个git用户;

添加证书登录

把刚在在本地创建或者已经拥有的公钥，也就是 ~/.ssh/id_rsa.pub 文件里的内容添加到服务器的 /home/git/.ssh/authorized_keys 文件中，如上所说，添加公钥之后可以防止每次 push 都输入密码。（*可以直接执行cat ~/.ssh/id_rsa.pub | pbcopy 复制）

hexo配置文件_config.yml蛋疼的空格问题...

看教程部署hexo到github

刚开始用notepad++编辑保存 “_config.yml”

运行hexo clean 提示 type位置出错,ok,试着删除type/repo/bbranch前面的2个空格:

hexo clean运行成功
然后运行 hexo -g 成功
再运行 hexo -d 什么提示都没有 运行失败
网上搜了搜 都是说什么冒号后面要有空格才行,我看看我的配置文件,冒号后面有空格啊,反复实验了n次仍然失败
想了想,莫非是保存文件的编码问题,把原来的hexo目录删除,重新hexo init了一次
这次用自带的vi编辑保存 “_config.yml”
因为前面的有空格出现错误,这次直接也删除掉type/repo/bbranch前面的2个空格
运行hexo -d 依然什么提示都没有 运行失败
实验了n次失败后,决定把type/repo/bbranch前面的2个空格加回去
运行hexo -d 这次成功了
